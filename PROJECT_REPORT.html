Deployment Instructions

  1. Ensure Java 24 is installed: C:\Program Files\Java\jdk-24\bin\java.exe
  2. Navigate to project directory: cd PatientAppointmentSystem
  3. Build the project: mvn clean package -DskipTests
  4. Run the application: "C:\Program Files\Java\jdk-24\bin\java.exe" -jar target/patient-appointment-system-1.0.0.jar
  5. Access the web interface: http://localhost:8080
  6. API documentation: http://localhost:8080/api/

Issues Identified & Resolved

Issue 1: Java Version Compatibility

Issue: UnsupportedClassVersionError - Java 8 runtime incompatible with Java 24 compiled code
Resolution: Configured system to use Java 24 runtime with proper JAVA_HOME settings
Status: RESOLVED ✓

Issue 2: API Context Path Routing

Issue: HTTP 404 errors on all API endpoints
Root Cause: Frontend calling /patients but backend mapping to /api/patients
Resolution: Created WebConfig for static resource serving, updated all controller request mappings with /api prefix
Status: RESOLVED ✓

Issue 3: Frontend API Communication

Issue: Health check endpoint misconfiguration, API prefix inconsistency
Resolution: Updated api.js with proper /api prefix for all 27 API methods
Status: RESOLVED ✓

Issue 4: Patient Form Validation

Issue: Failed to save patient - missing required backend fields
Resolution: Updated HTML form with Age and Medical History fields, fixed JavaScript to send proper JSON structure
Status: RESOLVED ✓

Key Design Patterns

  • MVC Pattern: Controllers handle HTTP requests, Models manage data, Views handled by API responses
  • Service Locator: Services encapsulate business logic and repository access
  • DAO Pattern: Repository interfaces abstract data access
  • DTOs: Data Transfer Objects separate API contracts from entities
  • Configuration Classes: Spring configuration for dependencies and beans
  • Exception Handling: Centralized exception management
